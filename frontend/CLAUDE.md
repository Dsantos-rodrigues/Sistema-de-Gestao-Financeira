## Project context

This is the **frontend** of FinFlow — an Open Finance wealth-management platform (gestão patrimonial). Sibling apps live next to it in the parent repo:

- `../backend` — NestJS API
- `../ia` — Python/FastAPI service for AI insights

FinFlow is **not** a personal expense tracker. It is a professional investor tool: portfolio analysis across BR equities, FIIs, foreign equities/ETFs and crypto; benchmark comparisons (CDI, Ibovespa); consolidated cash flow; and AI-driven recommendations with confidence scores. The UI must hide complexity and feel premium.

## Stack

- **Next.js 16.2.3** (App Router, Turbopack dev) — breaking changes vs 15.x; consult `node_modules/next/dist/docs/` before relying on memory.
- **React 19.2.4**
- **Tailwind CSS 4** via `@tailwindcss/postcss` — configured through the new `@theme inline` block in `app/globals.css` (no `tailwind.config.*` file).
- TypeScript, ESLint via `eslint-config-next`.

Next.js 16 routing details that often trip up code from older training data:
- `page` / `layout` `params` and `searchParams` props are `Promise<...>` and must be `await`ed.
- Global helper types `PageProps<'/route'>` and `LayoutProps<'/route'>` are auto-generated by `next dev` / `next build` / `next typegen` — no import needed.

## Commands

Run from `frontend/`:

```bash
npm run dev     # Dev server on :3000 (Turbopack)
npm run build   # Production build
npm run start   # Run the production build
npm run lint    # ESLint
```

No test runner is configured.

## Design system — "executivo sóbrio com calor dourado"

Financial seriousness without the sterile coldness of typical fintechs. Visual references:

- **Linear.app** — dark sidebar + light content area, clean typography, generous spacing, every pixel has purpose.
- **Stripe Dashboard** — KPI cards (small label on top, large value below), subtle borders, **no shadows**, status chips with same-hue colored bg + text.

Design tokens in `app/globals.css` via `@theme inline`, used as Tailwind utilities (`bg-gold-400`, `text-ink-900`, etc.):

- `gold-200..600` — warm gold accents (`#f1dca7` → `#a37e2c`). Avoid pure yellow; gold should read warm, not cold.
- `ink-700..900` — warm-leaning blacks for dark surfaces and primary text on light.
- Fonts: Geist Sans / Geist Mono via `next/font/google` (CSS vars `--font-geist-sans`, `--font-geist-mono`).

Rules to preserve when adding screens:

- Page background: light gray (`zinc-100`). Content surfaces: white with `border-zinc-200/80`, generous `rounded-3xl`, **no drop shadows**.
- Dark surfaces: `bg-ink-900` with subtle radial gold glows (low opacity, blurred) for warmth — never solid gold blocks.
- Primary CTA on light: `bg-ink-900 text-white`. Primary CTA on dark: `bg-gold-400 text-ink-900`.
- Inputs: `rounded-lg`, 1px border, `focus:ring-4` with low-opacity ring (`ring-zinc-900/5` on light, `ring-gold-400/10` on dark).
- Status / accent chips: low-opacity tinted bg + same-hue text + 1px inset ring (Stripe pattern).
